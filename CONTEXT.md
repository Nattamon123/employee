# Domain Glossary

## Systems & Actors
- **Employee App**: The native mobile application (Flutter / iOS / Android) used by employees for time attendance and submitting requests.
- **Admin App**: The web application (Vite + React/Vue) used by administrators or HR to monitor attendance, view logs, and approve requests.
- **API Gateway**: The central backend service written in Go (Gin) that handles all business logic, timestamping, and authorization.
- **Database**: Supabase (PostgreSQL), used strictly as a data store accessed by the API Gateway via `sqlx`/`pgx`.

## Core Concepts
- **Native Biometrics**: Identity verification using the device's built-in hardware (e.g., FaceID, Fingerprint). Proves that the person using the app is the device owner.
- **Device Binding**: A strict 1-to-1 association between an employee's account and a specific physical device identifier (UUID).
- **Unbind Device**: An administrative action performed by HR to clear a bound device from an employee's account, allowing them to log in on a new device.
- **Geofencing**: A virtual geographic boundary (e.g., a 50-meter radius around a Work Location). Check-ins outside this boundary are rejected by the API Gateway.
- **Work Location**: A named geographic point (lat/lng + radius) representing an office or work site. Admin can manage multiple Work Locations.
- **Mock Location**: Falsified GPS coordinates generated by third-party apps. The Employee App must detect and block attempts to use mock locations.
- **Offsite Request**: A formal request by an employee to work outside the office. When approved by HR, it temporarily disables Geofencing for that employee, allowing them to check in from any location (Trust-Based Offsite), while still logging their GPS coordinates for auditing.
- **Account Approval (Pending State)**: When an employee signs up using Google Auth, their account is placed in a "Pending" state. They cannot check in or access the app until an Admin explicitly approves their account.
- **Attendance Record**: A daily check-in/check-out entry for an employee. One record per user per day. Timestamps are set by the Go server, never by the client.
- **Late (สาย)**: Status applied when an employee checks in after 09:00.
- **Leave Request (ใบลา)**: A formal request for time off (sick leave, personal leave, swap holiday, work on holiday). Requires Admin approval.
- **Holiday (วันหยุด)**: A public or company holiday defined in the holidays table. Employees do not need to check in on holidays.
- **Role (สิทธิ์)**: Either `employee` or `admin`. Determines access level to API endpoints.
